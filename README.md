# Fontpub

> Identity-native font distribution. The verifiable protocol for open fonts.

## Manifest

Fonts, Reborn as Software.

Historically, fonts have been treated as static files—frozen the moment they are exported. While code has evolved into a dynamic ecosystem supported by version control and CI/CD, fonts have remained isolated in the "single file" era.

Fontpub bridges this gap. We bring the logic and lifecycle of software to the world of open fonts.

### 1. Direct Distribution, No Gatekeepers

Publishing a font should be as free as pushing to a Git repository. Fontpub rejects the idea that open fonts must pass through opaque, centralized gatekeeping before they can reach users. The repository is the source. Release is a public act. Distribution should flow from the work itself.

### 2. Provenance from the Canonical Source

We anchor trust in the repository itself. Fontpub ties published metadata to an immutable Git commit and a verifiable release workflow, so users can see where a font came from, what exactly was published, and what bytes they are installing. Trust moves away from opaque distribution silos and toward transparent source history and reproducible metadata.

### 3. From Static Files to Living Assets

A font is never truly "finished"—it evolves with bug fixes, new glyphs, and refined kerning. Fontpub treats fonts as versioned, inspectable, installable software: artifacts with history, integrity, and an update path. The result is not just easier distribution, but a new relationship between type designers, their repositories, and the systems where their fonts live.

### 4. Public Metadata, Minimal Black Boxes

Open infrastructure matters as much as open files. Fontpub is built around public, immutable metadata that can be inspected, mirrored, and rebuilt. What gets published should remain visible. What gets installed should remain verifiable. The protocol should not disappear behind a private database or a proprietary service boundary.

### 5. Tools for Humans and Machines

The future of software is collaborative, and that includes collaboration with tools. Fontpub is designed for designers, developers, and software agents alike: readable by humans, scriptable by machines, and structured enough to support automation without sacrificing clarity.
